# Camera System Implementation Plan

The goal is to ensure a robust camera system is fully functional before proceeding with other game features. Based on the current state of the codebase, these features are already integrated, so this plan will focus on verifying and refining them to guarantee they work perfectly.

## 1. Camera System Foundation
- Verify `src/camera/Camera.h` and `src/camera/Camera.cpp` properly encapsulate all camera properties (Position, Front, Up, Right, Yaw, Pitch).
- Ensure the view matrix is correctly calculated using `glm::lookAt`.

## 2. Mouse Control Integration
- Verify `mouse_callback` in `main.cpp` accurately captures cursor movement.
- Ensure the movement correctly updates the Camera's Yaw and Pitch.
- Verify pitch constraints (e.g., clamping between -89 and 89 degrees) to prevent gimbal lock or camera flipping.
- Verify `scroll_callback` correctly adjusts the camera's Zoom or Follow Distance.

## 3. Third-Person & FPS Camera Modes
- Ensure the `CameraMode` enum supports both `FPS` and `THIRD_PERSON`.
- Verify the toggle logic (e.g., pressing the `C` key in `main.cpp`) correctly switches between modes and prevents rapid toggling via debouncing.
- Ensure FPS mode places the camera exactly at/slightly above the target.
- Ensure Third-Person mode places the camera at a calculated distance and angle behind the target.

## 4. Camera Follow Logic
- Verify `Camera::FollowTarget` correctly updates the camera's position based on the ball's position.
- In Third-Person mode, ensure the camera uses the FollowDistance, FollowHeight, and SmoothSpeed to smoothly interpolate (e.g., using `glm::mix`) its position behind the ball, preventing jitter.

## Execution
Since the structural code for this is already present in `Camera.cpp` and `main.cpp`, execution will involve reviewing the existing implementation against these points, fixing any bugs (such as improper relative movement or camera snapping), and ensuring the game compiles and the camera works as requested.
